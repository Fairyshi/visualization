## 场景中的主要组件
组件名 | 描述
------------ | -------------
相机 | 决定哪些东西将在屏幕上渲染
光源 | 对材质如何显示，生成阴影时材质如何使用产生影响
物体 | 相机透视图中的主要对象，如平面、方块等

场景是Threejs中的主要容器，是渲染过程中所有物体、光源、相机的容器，主要是对上述组件进行管理。

### Scene的主要属性和方法
- add(object)
- children
- getChildByName(name)
- remove(object)
- traverse(function)
- fog[attr] 雾化
- overrideMaterial[attr] 可以让场景中所有的物体都使用相同的材质

## 自定义几何体Geometry
Geometry是三维空间中的点集及这些点连接起来的面，即 点集（vertices） + 面集(faces)。

建模时四边形 or 三角形？      
四边形是常采用的方式，因为它比三角形易增强和平滑。但对于渲染和游戏引擎来讲，使用三角形更容易，因为任何形状都可以渲染成多个三角形。

### 网格
考虑效率问题，Threejs库假设网格的几何体在其生命周期内不会改变。


## 相机
- 透视相机
尽量使用，近大远小，更贴近真实世界
- 正交相机
对象和相机的距离不会影响渲染效果，常用在二维游戏中。
